# feat-063 상점 구매 피드백

## 목표
상점에서 플레이어가 어떤 카드를 살 수 있는지, 왜 살 수 없는지, 구매 뒤 자금과 다음 전투 손패가 어떻게 되는지 바로 이해하게 한다.

## 범위
- 상점 카드마다 `구매 가능` 또는 `자금 부족` 상태 문구를 visible text로 표시한다.
- 구매 가능한 카드는 구매 후 남은 자금을 보여준다.
- 자금 부족 카드는 비용, 현재 자금, 부족 금액을 tooltip으로 표시한다.
- 구매 성공/실패 메시지는 `ShopPurchaseFeedback` 순수 helper로 계산한다.

## 비범위
- 상점 판매 목록, 가격, 카드 획득/보패 장착 로직은 변경하지 않는다.
- 구매 확인 모달, 환불, reroll, 상점 밸런스는 이번 피처에 포함하지 않는다.
- 전투 손패 refresh 규칙은 feat-058/060 계약을 그대로 사용한다.

## 검증
- 단위 테스트가 구매 가능, 자금 부족, 구매 성공, 구매 실패 문구를 검증한다.
- UI smoke가 상점 화면에서 구매 가능/자금 부족 문구와 구매 후 남은 자금 메시지를 확인한다.
- `./init.sh` 전체 검증을 통과한다.
