# feat-059 자동저장 슬롯 삭제 UX

## 목표
군주 선택 화면에서 저장된 런을 이어할지, 현재 자동저장 슬롯만 삭제하고 새 런을 시작할지 명확히 선택할 수 있게 한다. 손상 저장도 플레이어가 직접 지워 정상 시작 화면으로 복구할 수 있어야 한다.

## 범위
- 유효한 런 저장이 있으면 `저장된 런 이어하기` 아래에 `저장된 런 삭제` 버튼을 표시한다.
- 손상되었거나 호환되지 않는 런 저장이 있으면 기존 복구 안내 아래에도 `저장된 런 삭제` 버튼을 표시한다.
- 삭제 버튼은 런 autosave만 삭제하고, 프로필 해금과 기록은 유지한다.
- 삭제 뒤 군주 선택 화면을 다시 렌더해 이어하기/삭제 버튼이 사라진 상태를 보여준다.

## 비범위
- 다중 저장 슬롯, 삭제 확인 모달, 프로필 저장 삭제, 저장 포맷 변경은 이번 피처에 포함하지 않는다.
- 새 군주를 선택할 때 기존 런이 초기화되는 기존 동작은 유지한다.

## 검증
- 단위 테스트가 유효 저장과 손상 저장에서 삭제 버튼/tooltip 렌더를 확인한다.
- 저장/이어하기 UX smoke가 저장 런 삭제 후 이어하기/삭제 버튼이 사라지고 저장 파일이 제거되는지 확인한다.
- `./init.sh` 전체 검증을 통과한다.
