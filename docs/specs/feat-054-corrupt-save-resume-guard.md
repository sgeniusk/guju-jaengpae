# feat-054 손상 저장 이어하기 보호

## 목표
저장 파일이 존재하지만 손상되었거나 현재 버전에서 불러올 수 없을 때, 군주 선택 화면이 잘못된 이어하기 버튼을 보여주지 않도록 한다. 플레이어는 상태를 이해하고 새 군주를 골라 새 런으로 복구할 수 있어야 한다.

## 배경
feat-051은 저장된 런 이어하기 경로를 smoke로 고정했다. 하지만 현재 `has_run_save()`는 파일 존재만 확인하므로, 형식이 맞지 않거나 미래 major version 저장이 남아 있어도 이어하기 버튼을 노출할 수 있다. 완성판에서는 재시작 안정성이 중요하므로, “파일 존재”와 “로드 가능한 저장”을 분리해야 한다.

## 범위
- `RunManager.run_save_status()`를 추가해 저장 파일 없음, 로드 가능, 로드 불가 상태를 payload 검증까지 포함해 반환한다.
- `RunManager.has_resumeable_run_save()`는 실제로 `load_run()` 가능한 저장만 true를 반환한다.
- `lord_select.gd`는 로드 가능한 저장일 때만 이어하기 버튼을 표시한다. 로드 불가 저장이 있으면 비활성 안내를 표시하고, 새 군주 선택으로 새 런을 시작할 수 있게 한다.
- `run_map.gd`의 자동 로드도 로드 가능한 저장만 시도한다.
- `tools/resume_ux_smoke.gd`가 손상 저장 파일 케이스를 검증한다.

## 비범위
- 저장 포맷 변경이나 migration 추가.
- 손상 파일 자동 삭제.
- 슬롯형 저장 UI.
- 프로필 저장 손상 복구.

## 완료 기준
- 단위 테스트가 로드 불가 저장 파일의 status와 lord_select 안내를 검증한다.
- Resume UX smoke가 no-save, corrupt-save, valid-save 세 케이스를 모두 통과한다.
- `./init.sh` 전체가 green이다.
- 상태 파일과 changelog에 압축 증거를 남기고 로컬 커밋한다.
