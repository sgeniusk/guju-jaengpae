# feat-048 — 수동 플레이 QA 자동화

## 목표
첫 전투에서 플레이어가 직접 누르는 핵심 경로가 자동 검증에 묶이게 한다.

## 범위
- `tools/ui_feedback_smoke.gd`에 첫 수동 플레이 케이스를 추가한다.
- 스모크는 유비 런을 새로 시작하고 첫 손패를 계략, 병종, 건물 3장으로 고정한다.
- 플레이 순서는 성 타일 선택, 계략 카드 선택 후 타일 배치 거부 확인, 병종 카드 선택, 빈 타일 배치, 즉시 교전 시작이다.
- 교전 시작 후 성 위치, 보드 배치, 손패 감소, 교전당 1장 제한, 전투 phase, 성/아군/적 유닛 생성, rally hint를 검증한다.
- `./init.sh`의 기존 UI smoke 경로에 연결해 매 세션 검증에 포함한다.

## 비범위
- 카드 Resource 추가나 스키마 변경.
- 전투 밸런스, reward pool, 장기런 선택 정책 변경.
- 실제 마우스 좌표 기반 스크린샷 QA.
- 천계·마계 nation id 확장.

## 검증
- `tools/ui_feedback_smoke.gd`
- `./init.sh`
