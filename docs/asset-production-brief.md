# 구주쟁패(九州爭霸) — 전체 그래픽 애셋 풀 발주 브리프

> 외부 아트 에이전트(Manus 등)에게 게임 그래픽 **전체를 독립 풀세트로** 의뢰하는 자족 문서다. 저장소 접근 없이 이 문서만으로 작업한다. **기존 게임 내 임시(stopgap) 애셋과 무관하다 — 참조·정합 불필요. 아래 아트 디렉션으로 처음부터 통일성 있게 전부 새로 제작한다.** 모든 치수·경로·네이밍·팔레트는 게임 엔진 계약과 일치한다.

---

## 0. 게임 컨텍스트
- **제목** — 삼국지: 구주쟁패. **장르** — 풀 판타지 덱빌딩 로그라이크 **오토배틀러**(군주로 시작, 장수·병종 카드를 보드에 배치하면 자동 교전으로 적 파도를 막는다).
- **벤치마크** — *Nine Kings*(스팀). 좌측 본진(성·건물) + 우측에서 밀려오는 적 군세 + 중앙 난전 + 상단 스테이지 사다리 + 하단 체력바 + 떠오르는 데미지 숫자.
- **엔진** — Godot 4.x, **2D**. 애셋은 2D 스프라이트로 합성(3D 아님).
- **뷰/카메라** — 측면 살짝 부감한 횡전장. 유닛이 지면 위를 좌↔우로 행군해 중앙에서 교전. 16:9 (1920×1080).
- **무드** — 정사·연의 삼국지 + 봉신연의(封神演義)풍 동양 환상(신격 장수·요괴·선술). 비장하고 서사적.

## 1. 절대 준수 규격 (전 애셋 공통)

**1.1 포맷·배경** — **투명 배경 PNG(RGBA)** 가 기본. 생성기가 깨끗한 알파를 못 뽑으면 순수 마젠타 `#FF00FF` 단색(그라데이션·그림자 없이 가장자리까지 균일) 배경에 그린 뒤 크로마키로 제거해 납품. 배경에 접지 그림자를 굽지 말 것(엔진이 처리). 배경 이미지는 예외(§1.5).

**1.2 스타일 — 전 애셋을 하나의 스타일로 통일(이번 발주의 최우선 가치).**
- **기본 = 리치 디테일 픽셀아트.** *Nine Kings* 톤 — 정교한 픽셀, 손그림 느낌, 과한 굵은 외곽선 지양, 제한된 응집 팔레트, 또렷한 실루엣.
- (대안) 통일만 보장되면 고해상 2D 페인터리 게임아트도 가능. **반드시 단일 스타일로 전체를 일관**(픽셀·페인터리 혼용 금지).
- 광원 **좌상단 고정**, 채도·명도·외곽선·디테일 밀도를 전 애셋 동일.

**1.3 뷰·앵커 (엔진 합성 규칙 — 중요)**
- **유닛/장수/보스** — 전신, **측면 3/4 뷰로 화면 오른쪽을 향함**(아군 기준. 적은 엔진이 좌우 반전), **하단 중앙 = 발밑**(접지점), 상하좌우 여백 균일.
- **건물·성** — 정면 살짝 3/4, **하단 중앙 = 바닥**.
- **아이소 타일** — 2:1 다이아몬드, **중앙 앵커**.
- **UI 아이콘** — 정사각 중앙, 여백 일정.

**1.4 해상도** — 표의 "크기"는 게임 내 픽셀 기준. **2~4배 크게 생성 후 다운스케일**해 선명도 확보 가능(최종은 표 크기 또는 정수배, 비율 유지). 픽셀아트면 nearest 기준 깨끗하게(과한 안티앨리어싱 금지). 가능하면 1024px급 원본도 동봉.

**1.5 배경 예외** — 1920×1080 **불투명 PNG**. 인물·UI·텍스트 없이 빈 전장만. 아래 2/3 지면 + 위 1/3 하늘·원경, 중앙은 비워 유닛이 설 공간 확보.

```
assets/sprites/
├── bg/<theme>/field.png              # 1920×1080 불투명
├── iso/tile_*.png                    # 128×64 center
├── buildings/*.png                   # bottom-center
├── units/<faction>/<troop_type>.png  # 48×56 / 기병 64×56, 발밑 앵커, 오른쪽 보기

assets/fonts/pixel.ttf
faction ∈ { wei, shu, wu, kunlun, penglai, ziwei, huangtian, luoyang, wanyao }
```

## 5. 제작 워크플로 & 납품
1. **Phase 0 — 스타일 앵커(승인 게이트).** 먼저 대표 5종만 제작해 발주자 승인을 받는다 — ① 촉 장수 1(관우) ② 촉 보병 ③ 보스 마왕 동탁 ④ 성채 ⑤ UI 골드 아이콘. 스타일·팔레트·뷰·해상도 확정 후 양산.
2. **Phase 1~ — 티어 순 양산**(T0→T3, §6 순서).
3. **납품물** — ① 정확한 경로·파일명·치수·투명 알파의 개별 PNG ② 진영별 비교 **미리보기 시트**(한 장에 모아 일관성 확인) ③ 1024px 원본(가능 시) ④ 누락·대체 항목 회신.
4. **합격 기준** — 단일 통일 스타일·좌상단 광원·진영 팔레트 준수, 캐릭터 오른쪽 보기·발밑 접지·여백 일관, 진영 간 동일 병종 실루엣/스케일 일치.
