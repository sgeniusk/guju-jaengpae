# 설계 백로그 — 탐색 중 아이디어

확정 전 탐색 아이디어를 모은다. 확정되면 `docs/design-loop.md`(정본)·`feature_list.json`으로 승격한다. 확정 루프는 design-loop.md를 본다.

## 1. 전투 타겟 AI — 조건부 타겟팅 시스템 (가장 구체)
지금은 모든 유닛이 "최근접 적" 고정. 유닛(병종)마다 **공격 특성(타겟 규칙)**을 갖게 한다.
- 예시 규칙 — `nearest`(기본)·`strongest_ranged`(가장 센 원거리 우선)·`backline`(후열/먼 적)·`lowest_hp`(마무리)·`highest_hp`(탱커 견제)·`weakest`(약한 적).
- **데이터 주도** — 카드에 `target_rule` 필드, BattleSim이 규칙대로 표적 선택. 영웅 지정(commanded_target)은 이 규칙보다 우선.
- 효과 — 기병은 후열 다이브, 궁병은 센 원거리 견제 등 병종이 전술적으로 달라짐. "조건을 수용하는 시스템"이 바로 이것.
- 헤드리스 테스트 가능. 다음 시스템 후보 1순위.

## 2. 배치 = 기지(基地) 빌딩
"배치"는 전투 배치가 아니라 **기지 빌딩**을 말함. 스샷의 성벽 두른 기지 안에 건물을 짓는 것.
- NK에 아이디어 다수 — 건물 종류(소환·오라·경제), 기지 레이아웃, 성벽.
- 현재 단일 성 유닛(feat-014)을 건물 시스템(feat-016)으로 확장하면 그림이 됨.
- 배치 역할(전/후열)도 여기 연결 — 장수 앞, 궁병 뒤는 카드 속성.

## 3. 카드 분류 체계
- **영웅류(將帥)** = `general` — 조작 가능, 고유 스킬.
- **유닛류(兵種)** = `troop` — 타겟 규칙 따름.
- **책류(計略·주문)** = `scheme` — 1회성/지속 효과(책·주문).
- **보패(寶貝)** = `treasure` — 유물.
- **진영별 카드** — `nation` 필드로 이미 분류(촉·위·오·천계·마계 6국).
- **스페셜 카드** — 희귀/특수. `rarity` 필드 또는 special 플래그(신규).
- 현 스키마(general/troop/scheme/treasure + realm/nation)가 대부분 커버. 스페셜만 신규 필드.

## 4. 영웅 조작 연출
- 클릭 시 영웅 유닛이 표적으로 **가서 싸우는 모션**. (지금도 sim은 표적으로 이동하나, 시각적 강조·돌진 연출 추가.)
- 폴리시 항목. 시스템보다 비주얼.

## 우선순위 제안 (편집장)
1. **타겟 AI 시스템 (1)** — 가장 "시스템"스럽고 전투 깊이를 키움. 데이터 주도라 확장 쉬움.
2. **골드 경제 + 상점 (design-loop feat-015)** — 코어 루프 한 바퀴 완성.
3. **건물/기지 빌딩 (2, feat-016)** — 배치의 본체.
4. 카드 분류 확장(3)·연출(4) — 콘텐츠/폴리시로 점진.

> 메모 — 이 단계 핵심은 "확장 가능한 시스템" 깔기. 타겟 규칙·카드 종류·건물 효과를 데이터 주도로 만들면, 이후 콘텐츠(6국·스페셜·유물)는 데이터만 추가하면 된다.
